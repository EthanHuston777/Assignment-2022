Copyright, Creative Common (CC) Licenses, Fair Use

Copyright is a legal term which will protect a creator of almost all types from people stealing their creation. There is a wide range of materials under this law for example, books, music, journals, photographs and many more. Copyright is something that remains in place for several years, usually 95 years from publication or 120 years for the creation of work, and is automatically applied to works, even if the creator doesn’t do it themselves.

  • The first creative licencing is Attribute or BY. This symbol indicates you must attribute the source of work by siting the source.
  • The second creative licencing is called Share Alike or SA. This indicates that any work that you create using a work with this symbol must have the same creative licencing.
  • The third creative licencing means No Derivatives or ND. This indicates that the work its associated with cannot be changed when used but must stay the same.
  • The final creative licencing is Non-Commercial or NC. This indicates that any work using content with this symbol cannot be used to make money.
Fair use", or in NZ "fair dealing" is a way around copyright without asking for permission. This is the case when only a small amount is being copied or if the usage is research, private study, criticism or review or reporting current events. Read more at Copyright.co.nz

A privacy policy is a statement or document which declares how the user's/client's privacy will be maintained when they are using a service/product. It will explain to the user/client the actions that will be taken to ensure that no data/information is leaked, and what will happen if it is. It will also inform the user/client what their role and ability is in alignment with any of their information is. The privacy policy will also explain the use of the information/data they are handing over.

How To Improve Website Search Engine Optimization (SEO)

5 Ways To Improve Visibility Of A Website On Search Engine

  1. Crawl Accessibility – Giving engines the ability to read the website you have created.
  2. Compelling Content – Content that answers the searcher’s question/query.
  3. Keyword Targeting – Keywords are phrases commonly searched on different search engines, using them will help make your website more visible.
  4. Meta data – Putting keywords into your meta data (meta title and description) of a website will make it able to be matched easier through the search engine.
  5. Backlinks – This is to produce links from other websites back to your website so people may find it through a similar topic or accidentally find themselves there.

How To Chose Website Hoster/Publisher

3 Major Factors To Consider When Choosing A Website Hosting Provider

  1. Server Reliability – An important factor is how reliable the server hosting the website is, if it has poor reliability or if the service providers are unreliable then your money is not being well spent considering you’re not always getting the service.
  2. Account Limitations – Different hosting companies will allow a certain number of webpages and the number of scripts that are to be processed. This will determine how much you’ll be getting out of your website. This will need to be taken into higher consideration if your website is quite large or detailed.
  3. Price – How much the service will cost, through this alongside the above factors, you can determine if the website hosting service is worthwhile.

Web Performance & Maintenance

5 Ways To Reduce Page Loading Time

  1. Put script at the end of the page so that everything else (the actual website) loads and then the script.
  2. Choose a performance-optimized hosting solution (a hosting solution that runs quickly even on the fall of price.)
  3. Don’t scale down images - Resize the images with a software and implement into the website as a new image.
  4. Place JavaScript and CSS in external files - Keep them separate so they are loaded once when the website is opened for the first time and kept loaded, also allows for easier maintenance.
  5. Cache your web pages - statically cache wen pages to decrease strain on server as well as speed up page rendering times.

5 Important Actions To Carry Out When Maintaining A Website

  1. Test It Frequently - Run test to make sure website is working properly especially after updates to software/search engines.
  2. Update Regularly - Work on website code, allow for different disabilities and add accessibility options.
  3. Follow Tech Trends - Use the latest tech and code to have the best looking and feeling website possible.
  4. Run A Content Audit - To consistently produce high-performing content, keep track of content metrics and review your content.
  5. Review Website Performance - Watch amount of people entering website over time, update to keep as many as possible.

Web Security

5 Common Cyber Attacks & Their Defense

  1. Backdoor Trojan - An attack where the attacker makes their way through a back door and gains remote control of a victim's system.
  2. Cross-site scripting (XSS) attack - Attacker will frequently use JS for this attack, they will add malicious code to a proper website or app and use it to extract a user's information.
  3. Denial-of-service (DoS) - Overwhelming a system by flooding it, this attack is to break down the defences for another attack as the systems will be unable to respond or fight back.
  4. 4. DNS tunnelling - Cybercriminals use DNS tunnelling to exchange application data, they will setup false servers or attacked servers and make intact servers request data from them, silently and slowly giving the good servers malware or similar.
  5. Malware - Malware is malicious software that can render infected systems inoperable. Most malware variants destroy data by deleting or wiping files critical to the operating system's ability to run.

To protect against cyberattacks of any kind, including these, do the following. Create secure passwords and different passwords for different areas and application, you don't want the hacker to get everything. Have strong firewalls and similar systems in place to keep low level attacks at bay. Scan networks for anything suspicious and remove it immediately. Track suspicious DNS traffic by following dodgy or unheard-of names. Have deep scans always running and get some better protection than the default. Backup all drives into an impenetrable server that is isolated from the rest in case of an emergency. For all the above attacks, have a fail safe for them, like for a DDOS/DOS attack, have a system in place to catch items and lower the chances of being flooded.
