XML RULES


6 important rules that must be adhered to in order to create a valid XML file:

Rule 1: XML Documents Must Have a Root Element

A root element is simply a set of tags that contains your XML content. It can be called most things but the XML content must be within the root element.

Rule 2: All XML Elements Must Have a Closing Tag

When a tag is opened it needs to be closed to be counted as a completed tag otherwise the code will be broken, this applies for empty tags like line breaks like br and hr too.

Rule 3: XML Tags are Case Sensitive

This rule means that casing needs to be consistent in things like tags as they will create new tags if it is different.

Rule 4: XML Elements Must be Properly Nested

If one tag starts with another inside it, then the tag inside needs to be completed before the incasing tag.

Rule 5: XML Attribute Values Must Always be Quoted

Tag names are not allowed to begin with a number, punctuation, or "xml", however they can begin with an underscore character "_".

Rule 6: Tag Names Have Strict Limits

In XML, all attribute values must appear within quotes.
